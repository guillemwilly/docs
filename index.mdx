**Agno is software for building agent platforms.**

- Build agents using any agent framework.
- Run them as production services with session management, tracing, scheduling, and RBAC.
- Manage your platform using a single control plane.

<CodeGroup>

```python Agno SDK
from agno.os import AgentOS
from agno.agent import Agent
from agno.db.sqlite import SqliteDb
from agno.tools.workspace import Workspace

agent = Agent(
    name="Agno Agent",
    model="openai:gpt-5.4",
    tools=[Workspace(".",
        allowed=["read", "list", "search"],
        confirm=["write", "edit", "delete", "shell"],
    )],
)

agent_os = AgentOS(
    agents=[agent],
    tracing=True,
    db=SqliteDb(db_file="tmp/agentos.db"),
)
app = agent_os.get_app()
```

```python LangGraph
from agno.agents.langgraph import LangGraphAgent
from agno.db.sqlite import SqliteDb
from agno.os import AgentOS
from langchain_openai import ChatOpenAI
from langgraph.graph import MessagesState, StateGraph

def chatbot(state: MessagesState):
    return {"messages": [ChatOpenAI(model="gpt-5.4").invoke(state["messages"])]}

graph = StateGraph(MessagesState)
graph.add_node("chatbot", chatbot)
graph.set_entry_point("chatbot")
compiled = graph.compile()

agent = LangGraphAgent(
    name="LangGraph Chatbot",
    graph=compiled,
)

agent_os = AgentOS(
    agents=[agent],
    tracing=True,
    db=SqliteDb(db_file="tmp/agentos.db"),
)
app = agent_os.get_app()
```

```python DSPy
import dspy
from agno.agents.dspy import DSPyAgent
from agno.db.sqlite import SqliteDb
from agno.os import AgentOS

dspy.configure(lm=dspy.LM("openai/gpt-5.4"))

agent = DSPyAgent(
    name="DSPy Assistant",
    program=dspy.ChainOfThought("question -> answer"),
)

agent_os = AgentOS(
    agents=[agent],
    tracing=True,
    db=SqliteDb(db_file="tmp/agentos.db"),
)
app = agent_os.get_app()
```

20 lines of code and you have a FastAPI backend with 50+ endpoints, persisted sessions, tracing, scheduling and RBAC. Interact using the AgentOS UI, Slack, Discord or build a product on top.

Run native Agno agents next to Claude Code, LangGraph and DSPy agents in the same AgentOS. See [Multi-Framework Support](/agent-os/multi-framework/overview).

## Three layers, one architecture

Everything except the control plane is free and open-source.

| Layer | Use it to | What it does |
|-------|------|--------------|
| **SDK** | Build | Agents, multi-agent teams, and agentic workflows with memory, knowledge, guardrails, and 100+ integrations. |
| **Runtime** | Run | A stateless, secure FastAPI backend with sessions, scheduling, and interfaces. |
| **Control Plane** | Manage | Operate your platform from the [AgentOS UI](https://os.agno.com). |

## Agno features

- [**Production API.**](/runtime/serve-as-api) 50+ endpoints with SSE and websockets to build a product on top of your agent platform.
- [**Storage.**](/runtime/storage) Sessions, memory, knowledge, and traces in your own database. Postgres for sessions and memory, ClickHouse for OLAP traces.
- [**100+ integrations.**](/tools/toolkits/overview) Pre-built toolkits for the systems your agents need to reach.
- [**Context Providers.**](/runtime/context) Live data from Slack, Drive, wikis, MCP, and custom sources.
- [**Human approval.**](/runtime/human-approval) Pause runs for user confirmation, admin approval, or external execution.
- [**Observability.**](/runtime/observability) OpenTelemetry tracing, run history, and audit logs out of the box.
- [**Security.**](/runtime/security-and-auth) JWT-based RBAC and multi-user, multi-tenant isolation.
- [**Interfaces.**](/runtime/interfaces) Slack, Telegram, WhatsApp, Discord, AG-UI, A2A.
- [**Scheduling.**](/runtime/scheduling) Cron-based scheduling and background jobs without external infra.
- [**Deploy anywhere.**](/runtime/deploy) Docker, Railway, AWS, GCP. Any container host works.
